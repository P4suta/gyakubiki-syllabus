# 逆引きシラバス — プロジェクト概要

高知大学の公開シラバスを「曜日 × 時限」の時間割グリッドで逆引きする、個人制作の非公式ツール。

- **公開先**: <https://p4suta.github.io/gyakubiki-syllabus/>
- **ソース**: <https://github.com/P4suta/gyakubiki-syllabus>（AGPL-3.0）
- **収録**: 2026年度 約3,900科目

このドキュメントは、本プロジェクトを短時間で把握するための要約。技術詳細は
[`kulas-api-spec.md`](kulas-api-spec.md)（取得仕様）と [`testing.md`](testing.md)（テスト戦略）に分けてある。

## 何ができるか

| 機能 | 内容 |
|---|---|
| 逆引き | 曜日 × 時限のグリッドから、その枠に開講されている科目を引く |
| 学期切替 / 学部フィルタ | 上部タブとセレクトで絞り込む |
| 検索 | 科目名・教員名・科目コードに加え、シラバス本文（概要・キーワード・到達目標）を全文検索 |
| 科目詳細 | 成績評価の内訳（円グラフ）、授業計画、到達目標、教科書、オフィスアワー |
| マイ時間割 | 科目をグリッドに登録すると、単位数が合計され、時間割の重複を自動検出 |
| 共有URL | 組んだ時間割をURLに符号化して共有 |
| 公式リンク | 各科目から公式KULASのシラバスページへ直接遷移 |
| オフライン | PWA。Service Worker でシェルをキャッシュ |

「公式シラバスの置き換え」ではなく「公式への入口」として設計している。詳細画面には常に公式KULASへの導線がある。

## なぜハルシネーションが起きないか

**生成AIを一切使用していない。**

公開シラバスを Rust で機械的に変換し、そのまま再表示している。処理は決定論的で、
同じ入力からは常に同じ出力が出る。文章を生成する箇所が存在しないため、
事実と異なる内容が混入する経路がない。

この性質は「独自解釈を挟まない」という方針として徹底している。たとえば科目コード
`72136`（卒業研究）は、KULAS が担当教員名を重複させた状態で返してくるが、
**重複を除去せずそのまま表示している**。読みやすさより公式との一致を優先しており、
表示がおかしい場合は原則として元データがそうなっている。

## どう作られているか

| 層 | 技術 |
|---|---|
| データ変換・検索エンジン | Rust（`crates/core`） |
| ブラウザ実行 | WebAssembly（`crates/wasm`、wasm-bindgen） |
| 取得・変換CLI | Rust（`crates/cli`） |
| UI | Svelte 5 + Vite + Tailwind CSS |
| 配信 | GitHub Pages（静的）+ GitHub Actions |

検索エンジン本体は Rust で実装し、WASM 経由でブラウザ内で動かしている。
サーバを持たず、約1MBのデータをクライアント側に置いて全文検索する構成のため、
**大学側にも作者側にも稼働中のサーバが存在しない**。詳細データは科目単位に分割し、
モーダルを開いた時点で遅延読み込みする。

品質面では、Rust / Web 双方のカバレッジゲート、プロパティベーステスト、
DOMジオメトリテスト、ビジュアル回帰、アクセシビリティ（axe による WCAG 2 A/AA の自動検証）、
ミューテーションテスト（cargo-mutants / Stryker）、ファジング、CodeQL を CI に組んでいる。
詳細は [`testing.md`](testing.md)。

## 大学に負荷をかけない設計

取得対象は、**ログイン不要（GUEST）で一般に公開されているシラバス情報のみ**。
非公開情報には一切アクセスしていない。その上で、アクセスは「隠れないこと」を
方針の中心に置いている。全文は [Politeness / responsible access](kulas-api-spec.md#politeness--responsible-access)。

| 項目 | 実装 |
|---|---|
| 身元の明示 | User-Agent に `gyakubiki-syllabus/1.0 (+<リポジトリURL>)` を付加。気づいた運用者が連絡できるようにするため |
| 隠蔽の拒否 | IPローテーション等を意図的に行わない。単一の GitHub Actions runner のみ。予測可能で連絡可能な単一の発信元であること自体が目的 |
| レート制限 | 詳細クロールは完全逐次（並列なし）、1件ごとに3〜5秒のジッター付きスリープ |
| 後退 | 指数バックオフ、`Retry-After` の尊重、連続拒否時はサーキットブレーカーで実行全体を中断（ブロックがハンマリングに転化しない） |
| 総量 | 更新された科目のみ再取得（増分）、1回あたりの件数上限、日本時間の早朝に実行 |
| 頻度 | 基本情報は季節可変（3・4・9・10月は日次、他は週次）、詳細は日次だが通常は取得0件で終了 |
| robots.txt | `kulas.kochi-u.ac.jp/robots.txt` は 404（2026-07-07 確認）。RFC 9309 上、robots.txt 不在は制限の宣言なしを意味するため、遵守すべき `Disallow` が存在しない |
| ローカル取得の禁止 | KULAS へのアクセスは GitHub Actions からのみ行う運用とし、開発時は KULAS に触れずに済む合成データ経路（`just gen-sample` / `just dev-sample`）を用意 |

## 立ち位置

- **非公式**。高知大学とは一切の関係がなく、同大学の承認または推奨を受けたものではない
- **公開情報のみ**に基づく。非公開情報は使用していない
- **AGPL-3.0**。ソースは公開されており、改変版をネットワーク越しに提供する場合も同ライセンスでのソース公開を要する（囲い込みを防ぐため）
- 履修登録その他の判断は、必ず大学公式の情報に基づいて行うこと。詳細は
  [README の Disclaimer](../README.md#disclaimer)

### 既知の論点

事実データ（曜日・時限・科目名・教員名・科目コード）は著作物性が低い一方、
**シラバス本文（授業概要・到達目標・授業計画）は執筆した教員の著作物**であり、
本ツールはその再配布について許諾を得ていない。

本文表示を抜粋に留めて公式KULASへ誘導する形に変更することは可能で、その場合も
中核である「曜日 × 時限の逆引き」の機能は維持される。
